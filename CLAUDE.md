### The Final Fix:
**Run this SQL in Supabase SQL Editor**:
```sql
CREATE OR REPLACE FUNCTION match_documents(
  query_embedding text,  -- Changed from vector to text
  match_model_id uuid,
  match_count int DEFAULT 5,
  similarity_threshold float DEFAULT 0.7
)
RETURNS TABLE (
  id uuid,
  training_data_id uuid,
  chunk_text text,
  chunk_index int,
  similarity float
)
LANGUAGE plpgsql
AS $$
BEGIN
  RETURN QUERY
  SELECT
    de.id,
    de.training_data_id,
    de.chunk_text,
    de.chunk_index,
    1 - (de.embedding::vector <=> query_embedding::vector) AS similarity
  FROM document_embeddings de
  WHERE de.model_id = match_model_id
    AND 1 - (de.embedding::vector <=> query_embedding::vector) >= similarity_threshold
  ORDER BY de.embedding::vector <=> query_embedding::vector
  LIMIT match_count;
END;
$$;
```

users (

  base_model TEXT,
  config JSONB,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)

branding_config (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  domain TEXT UNIQUE,
  logo_url TEXT,
  primary_color TEXT,
  secondary_color TEXT,
  company_name TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP

training_data (

chat_sessions (

  updated_at TIMESTAMP
)

chat_messages (

  created_at TIMESTAMP,

waitlist (

## Troubleshooting Common Issues

### Issue 1: Supabase Connection Fails
```
Check:
‚ñ° Environment variables are set correctly
‚ñ° Supabase project is not paused
‚ñ° API keys have correct permissions
‚ñ° CORS is configured in Supabase dashboard
‚ñ° Network connection is stable

Fix: Verify .env.local and restart dev server
```

### Issue 2: Ollama Not Responding
```
Check:
‚ñ° Ollama service is running (systemctl status ollama)
‚ñ° Model is downloaded (ollama list)
‚ñ° Port 11434 is not blocked
‚ñ° Firewall allows local connections

Fix: Run 'ollama serve' and 'ollama pull mistral'
```

### Issue 3: File Upload Fails
```
Check:
‚ñ° File size under limit
‚ñ° File type is allowed
‚ñ° Supabase Storage bucket exists
‚ñ° RLS policies allow upload
‚ñ° User is authenticated

Fix: Check Supabase Storage permissions and policies
```

### Issue 4: White-label Branding Not Loading
```
Check:
‚ñ° Domain is in branding_config table
‚ñ° Logo URL is accessible
‚ñ° Colors are valid hex codes
‚ñ° Cache is cleared
‚ñ° Middleware reads headers correctly

Fix: Check domain matching logic in middleware
```

### Issue 5: Stripe Webhook Not Working
```
Check:
‚ñ° Webhook endpoint is accessible
‚ñ° Webhook secret matches Stripe dashboard
‚ñ° Stripe CLI is forwarding events (development)
‚ñ° Signature verification passes

Fix: Use Stripe CLI 'stripe listen --forward-to localhost:3000/api/stripe/webhook'
```
